Call Flow Logic

        
Step-by-step routing for office hours and after-hours calls


        

          

            

              
Office Hours Flow

              Active
            

            

              
1
Greeting
"Thank you for calling Ben's Electric Solutions. This is Clara, how can I help you today?"

              
2
Understand Purpose
Listen to caller's reason. Do not interrupt.

              
3
Collect Name & Number
"May I get your name and the best number to reach you?"

              
4
Transfer to Ben
"Let me connect you now, please hold." → Initiate transfer to Ben's number.

              
5
If Transfer Fails
"I'm sorry, I wasn't able to connect you. I've noted your details and Ben will follow up soon."

              
6
Pricing (if asked)
$115 callout fee + $98/hr in 30-min increments. Only mention if caller asks.

              
7
Close
"Is there anything else I can help you with?" → "Thank you for calling. Have a great day!"

            

          


          

            

              
After-Hours Flow

              Restricted Emergency
            

            

              
1
Greeting
"Thank you for calling. Our office is currently closed. Mon–Fri, 8:30–5:00. How can I assist you?"

              
2
Understand Purpose
Listen to caller's reason.

              
3
Confirm Emergency
"Is this an emergency situation that requires immediate assistance tonight?"

              
4a
IF Emergency + G&M Client
Collect name, phone, property address immediately → Transfer to Ben → If fails: assure urgent callback.

              
4b
IF Emergency + Other Caller
"We don't have after-hours coverage for new calls. I've noted your details — someone will reach out first thing in the morning."

              
4c
IF Not Emergency
Collect name, number, brief description → "We'll follow up with you next business day."

              
5
Close
"Is there anything else I can help you with?" → Thank them, close warmly.

Run Pipeline

        
Process a new transcript or onboarding update


        

          

            

              
Pipeline A — New Account (v1)

              Demo / Onboarding
            

            

              Upload a demo call transcript or onboarding call transcript to generate a new v1 account memo and agent spec.
            

            

              
📄

              
Drop transcript here

              
.txt, .docx, or .md — max 10MB

              
            

            

            

              

                
Source Type

                
              

            

            

              
Command

              
python3 scripts/local_pipeline.py \
  --transcript data/<file>.docx \
  --source-type onboarding_call

            

          


          

            

              
Pipeline B — Update Account (v2)

              Onboarding Update
            

            

              Upload onboarding call or form data to patch an existing v1 account and generate v2 with changelog.
            

            

              
🔄

              
Drop onboarding update here

              
.txt, .docx, or .md — max 10MB

            

            

              
Command

              
python3 scripts/local_pipeline.py \
  --transcript data/onboarding_<account>.docx \
  --pipeline B \
  --v1-memo outputs/accounts/<id>/v1/account_memo.json

            

          

        


        

          

            
Batch Runner

          

          
Process all transcripts in a dataset folder at once.

          
# Process all accounts in dataset folder
python3 scripts/batch_run.py --dataset ./data

# Process a single account
python3 scripts/batch_run.py --dataset ./data --account bens-electric-solutions

# With Claude API (optional — for better extraction)
ANTHROPIC_API_KEY=your_key python3 scripts/local_pipeline.py --transcript ...

        


        

          
Dataset File Naming Convention

          

            
Name your transcript files as follows for the batch runner to pair them correctly:

            
data/
  demo_acme-fire-protection.docx        ← Pipeline A (demo)
  onboarding_acme-fire-protection.docx  ← Pipeline B (onboarding update)
  demo_metro-hvac.txt
  onboarding_metro-hvac.txt
  some-company.docx                     ← treated as onboarding-only
